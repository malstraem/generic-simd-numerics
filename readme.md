This is prototype of generic linear algebraic objects using current state of
[System.Runtime.Intrinsics](https://learn.microsoft.com/ru-ru/dotnet/api/system.runtime.intrinsics?view=net-10.0) SIMD vectors.

Once was started as try to outperform [Silk.NET](https://github.com/dotnet/Silk.NET), especially on matrix operations with `double` precision.

This stuff is made primarily for in-house engine, but it's possibly helpful draft for https://github.com/dotnet/runtime/issues/24168

### Achieved:

- Acceleration for `Vec4<T>`, `Quat<T>`, `Rect<T>` and `Mat44<T>` - operations, various methods and type casts with usefull type combinations
are effectively accelerated (with couple of unnecesary instructions generated by JIT in some places)

- All structs constrainted by `INumber<T>` except of `Quat<T>`

- Unlike `Silk.NET.Maths` and `System.Numerics`, matrix 4x4 have accelerated method
to construct affine matrix from quaternion, scale and translation vectors

- Unlike `Silk.NET.Maths`, rectangle is defined as origin+max to effectively accelerate intersects/contains methods

### Not achieved:
- Acceleration for `Vec2<T>` and `Vec3<T>` - now there is no way to generate effective load/store operations for structs
that not aligned to naturally register size ([see](./reports/vec2-load-store-comparison.md))

> [!NOTE]
check [benchmark reports](./reports) for generated asm and timings</br></br>
tested and benchmarked on AMD Ryzen 9 7900X that have</br></br>
`AVX512 - BITALG+VBMI2+VNNI+VPOPCNTDQ+IFMA+VBMI+F+BW+CD+DQ+VL`</br></br>
`AVX2 - BMI1+BMI2+F16C+FMA+LZCNT+MOVBE`</br></br>
`AVX+SSE3+SSSE3+SSE4.1+SSE4.2`
